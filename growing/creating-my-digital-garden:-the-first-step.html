Content

I came across the concept of digital gardening randomly one day while scrolling through Youtube. I found the concept really interesting and wanted to explore further. I read a few articles to understand it completely and also wrote an article myself. But I didn't actually made my own digital garden till now. Here I am after 3 month working on this again because this concept never truly left my mind. The idea of growing your thoughts and ideas bit by bit and in public was interesting and I want to give it shot. This is the first version of something bigger I'm planning to create in the next few months.

Technical Step-by-Step process until now:

1. Choosing tech stack

I wanted to keep things simple and lightweight, so I went with:

  • HTML + Tailwind CSS (for structure and styling)
  • JavaScript (for interactivity)
  • Node.js (for scripting and automation (file handling, JSON updates))

2. Setting Up the Project Structure

I organised my folder structure as shown in the screenshot:

  • posts.json - central store of metadata about all posts (title, status, dates, content, etc.)
  • admin.js - script for creating new posts from templates
  • edit.js - script for syncing post content back into posts.json

3. Creating the Templates

Inside the template/ folder, I built starter HTML files for each category (seed, growing, harvested, abandoned).

4. Building the admin.js Script

The goal of admin.js is to automate adding new posts.

  • Takes a template (based on category).
  • Creates a new HTML file in the right folder.
  • Updates posts.json with metadata: title, status, createdDate, lastUpdated, content.

This way, I donâ€™t need to manually edit both HTML and JSON whenever I add something new.

5. Building the edit.js Script

The goal of edit.js is to sync edits made in HTML files back into posts.json.

  • Reads through all folders (seeds/, growing/, etc.).
  • Extracts HTML inside 
    <div id="content">...</div>
  • Updates the matching entry in posts.json.
  • Refreshes lastUpdated.

That way, I can freely edit my notes in HTML while keeping posts.json as the single source of truth. I used Cheerio (a Node.js library for HTML parsing, like jQuery for the server) to make extracting and updating the content straightforward.

6. Why Iâ€™m Using posts.json Instead of a Database

I decided to use a JSON file instead of setting up a full database because:

  • Simplicity - I donâ€™t need a server or database running in the background.
  • Version Control Friendly - JSON works great with Git, so every change is tracked.
  • Good Enough for Now - A digital garden doesnâ€™t need heavy DB features; JSON is fast for a few dozen posts.

When the collection grows and crosses around 100 posts, Iâ€™ll port everything to a proper database.

7. Why I Run Scripts Locally Instead of Making a UI

I could have built a full admin panel, but I intentionally chose to run admin.js and edit.js manually:

  • Control - I can run them whenever I want, without extra UI complexity.
  • Focus on Content - I donâ€™t want to spend weeks building an editor when my goal is to write and grow ideas.
  • Learning Purpose - Running Node.js scripts teaches me file handling and parsing in a clean way.
  • Flexibility - I can later build a UI on top of this system if I ever feel the need.

8. Keeping index.html Updated

The index.html file serves as the map of my garden. It groups posts by category and uses posts.json as the source of truth for links and metadata.

9. Workflow I Follow

  • Run admin.js - create a new note from template.
  • Edit the HTML content - write and style freely with Tailwind.
  • Run edit.js - sync back into posts.json.
  • Open index.html - view the updated garden.
